# 多-Agent 向单进化 Agent 收敛的结构调整方案

> 方向：从“多主脑协作”收敛到“单主脑 + 功能型分身/执行器”。
> 原则：主脑唯一、记忆集中、分身短命、输出统一。

---

## 1. 核心判断

当前多-agent 结构更多是在补：
- 单模型长流程不稳
- 规划/执行/审查难同时做好
- 长任务易漂移
- 工具链复杂时容易乱

这意味着：
- 多-agent 当前更像工程补偿
- 单进化 agent 才更像长期主线

因此结构上应该开始收敛。

---

## 2. 目标结构

### 从现在的倾向
- 多个 agent 都像主脑
- 多个节点都可能做规划/判断/输出
- 记忆可能分散
- trace 责任面变模糊

### 调整到
- **一个主脑 agent**：唯一长期主体
- **若干功能型分身**：执行器 / 顾问器 / 并行 worker
- **长期记忆集中在主脑**
- **最终输出统一从主脑发出**

---

## 3. 推荐新结构

### A. 主脑层（唯一）
建议：`Sage / Mindclaw 主 agent`

职责：
- 统一人格 / SOUL
- 统一记忆
- 统一策略判断
- 统一风险决策
- 统一任务编排
- 统一最终输出

要求：
- 只有一个长期主身份
- 长期经验只沉淀在这里
- 不再与其他 agent 平级共享“主脑权”

### B. 执行层（Executor）
如：
- Codex Executor
- Browser Executor
- File Executor
- Retrieval Worker

职责：
- 接具体任务
- 做机械执行
- 回传结果

要求：
- 不保留长期人格
- 不持有长期记忆
- 不独立决定长期策略

### C. 顾问层（Advisor）
如：
- 架构顾问
- 复杂任务规划器
- 风险复核器

职责：
- 提供计划 / 建议 / 中途纠偏
- 不直接拥有高风险执行权

要求：
- 只给 guidance
- 不绕过主脑和 RiskGate

### D. 并行层（Worker）
适用于：
- 多源检索
- 多文件扫描
- 批量分析
- 多候选并行验证

要求：
- 临时拉起
- 无长期状态
- 做完即回收

---

## 4. 关键调整点

### 4.1 取消“平级多主脑”
如果多个 agent 都能：
- 规划
- 记忆
- 输出
- 调度别人

那就会导致：
- 记忆分裂
- 决策冲突
- trace 难看
- 责任不清

应调整为：
- 主脑唯一
- 其他 agent 功能化

### 4.2 长期记忆集中
应统一到主脑：
- 用户偏好
- 项目上下文
- 成功经验
- failure lessons
- candidate / learned skills

执行器只保留：
- 当前任务上下文
- 临时缓存
- 输出结果

### 4.3 调度统一从主脑发起
不再让 agent 之间互相无限调度。

建议模型：
```text
用户
 → 主脑
   → 判断是否调用执行器/顾问器/worker
   → 收集结果
   → 自己做最终决策与输出
```

### 4.4 输出统一
最终用户看到的结果，应尽量统一由主脑发出。

这样能保证：
- 风格一致
- 记忆连续
- 责任清晰
- trace 容易看

---

## 5. 对当前组件的映射建议

### 保留为主脑
- Sage
- Mindclaw Runtime

### 收敛为执行器
- Codex → `Codex Executor`
- 浏览器相关 → `Browser Executor`
- 文件修改链路 → `File Executor`
- 搜索/检索 → `Retrieval Worker`

### 收敛为顾问器
- 高级规划能力
- Advisor / Planner（如后续接 advisor tool）

---

## 6. 调整优先级

### P0
- 明确唯一主脑是谁
- 禁止新增平级主脑

### P1
- 把现有其他 agent 改成 executor / advisor / worker 角色
- 去掉它们的长期人格定位

### P2
- 收走长期记忆
- 统一沉淀到主脑

### P3
- 主脑统一编排所有子单元
- 子单元只回结果，不直接对外主导输出

---

## 7. 长期价值

这样调整后，系统会更接近：
- 单主体持续成长
- 经验可积累
- 失败可归因
- 纠偏更自然
- 成本更可控
- trace 更清晰

这比“多 agent 永久平级协作”更适合走长期演进路线。

---

## 8. 一句话结论

多-agent 当前更多是阶段性补偿。
后续应收敛到：

**单主脑 + 功能型分身 + 集中记忆 + 统一输出**

这才更像单进化 agent 的长期结构。
